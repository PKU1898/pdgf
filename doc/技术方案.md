2. 架构设计
整体架构图（文字描述）

  ↓ (HTTPS/JSON)

  ↓ (Prisma ORM)
[PostgreSQL 数据库] (存用户、图纸、库存关系)
  ↓ (外部API调用)
[第三方服务] (对象存储OSS存原图、Remove.bg做AI抠图、微信支付)
核心技术诀窍（非程序员必看）：
不要用小程序的 Canvas 做画板编辑！ 小程序 Canvas 坑极多，AI也写不好。拼豆天然是网格，直接用 CSS Grid 渲染一堆 <view> 标签，每个 view 就是一颗豆。点击改变背景色即可。这样开发速度极快，AI生成准确率100%。只在最后“导出图片”时，才把 Grid 数据画到 Canvas 上生成PNG。
3. 数据库Schema (Prisma Schema)
这是AI最熟悉的Prisma格式，天然包含关联关系和索引：
datasource db {
  provider = "postgresql"
  url      = env("DATABASE_URL")
}
generator client {
  provider = "prisma-client-js"
}
model User {
  id           String         @id @default(cuid())
  openId       String         @unique // 微信OpenID
  nickname     String?
  vipStatus    Int            @default(0) // 0免费 1订阅 2买断
  projects     Project[]
  inventories  Inventory[]
  createdAt    DateTime       @default(now())
}
model Project {
  id           String         @id @default(cuid())
  userId       String
  name         String
  width        Int            // 画板宽
  height       Int            // 画板高
  brand        String         // 色卡品牌 mard/coco
  gridData     Json           // 核心图纸数据：二维数组存色号ID
  user         User           @relation(fields: [userId], references: [id])
  createdAt    DateTime       @default(now())
  updatedAt    DateTime       @updatedAt
  @@index([userId]) // 按用户查询图纸加速
}
model BeadColor {
  id           String         @id // 如 "mard_024"
  brand        String         // 品牌
  code         String         // 色号
  hexCode      String         // #FF0000
  rgb          String         // 255,0,0 (用于计算色差)
  name         String?        // 中文名
  @@unique([brand, code]) // 同品牌色号唯一
}
model Inventory {
  id           String         @id @default(cuid())
  userId       String
  colorId      String         // 关联 BeadColor 的 id
  quantity     Int            // 库存数量
  user         User           @relation(fields: [userId], references: [id])
  @@unique([userId, colorId]) // 一个用户同色只有一条记录
  @@index([userId])
}

6. 关键技术决策
前端推荐 Uni-app (Vue3)：Vue3的模板语法对非程序员最直观，且AI工具对Vue3的补全和纠错能力远超其他框架，能帮你避开90%的小程序兼容性坑。
后端推荐 Node.js (Express+Prisma)：前后端统一用Javascript，你只需要学一种语言；Prisma让数据库操作变成写对象属性，不需要写任何SQL语句，AI生成Prisma代码的准确率接近100%。
数据库推荐 PostgreSQL：免费且性能强劲，它的JSONB数据类型完美契合“图纸网格数据”这种动态二维数组，避免了复杂的表关联设计。
部署推荐 Render：一键关联GitHub仓库，自动部署Node后端和Postgres数据库，零运维成本，免费额度完全够MVP验证。
画板渲染推荐 CSS Grid：放弃Canvas编辑，把豆豆当成DIV网格渲染，极大降低前端状态同步难度，AI写CSS Grid又快又好。
7. 风险预案
技术风险	发生概率	影响	应对方案
图片像素化算法内存溢出	中	高	限制上传图片大小（<5MB），在后端使用 sharp 库流式处理图片，而不是全部读入内存；设置API超时熔断。
CSS Grid 渲染 100x100 豆豆卡顿	低	高	超过 50x50 时，采用虚拟列表技术（只渲染视口内的DOM），AI有现成的虚拟滚动组件可抄。
Render 免费服务冷启动慢	高	低	休眠后首次唤醒需20秒，可用第三方免费监控工具（如 UptimeRobot）每5分钟ping一次后端，保持常醒。
色差计算导致后端CPU跑满	中	高	映射库存色卡时，如果图纸超过2000颗豆，改为异步任务计算，前端轮询结果，避免API阻塞超时。
