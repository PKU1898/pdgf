1. 产品概述（≤100字）
一句话定位：
面向拼豆玩家的“图纸 + 库存 + 采购”一站式小程序工具。
核心价值主张：
上图即图纸、自动算豆、库存配色、缺货提醒、导购下单，从选图到买豆一步完成。
与竞品核心差异：
竞品多只做到“生成图纸 + 统计数量”，我们叠加 个人豆库 + 余量计算 + CPS 电商闭环，解决“买了豆不对、拼到一半缺色”的深层痛点。

验收标准：点击单颗豆，弹出色板选择新颜色，该点立即更新且统计重新计算。
作为玩家，我希望框选区域一键填充/去色，以便快速上色或擦除背景。
验收标准：选中区域后点击“填充/去色”，区域内所有格子按规则统一处理。

验收标准：修改宽高后画布自适应，已有内容尽量保留（缩小裁边需二次确认）。

验收标准：去噪/融合后，孤立单点和细碎过渡明显减少，但主体轮廓不被破坏。

3.1 图片一键转图纸（P0）
描述：上传图片，自动像素化并映射到当前品牌色卡，生成可编辑图纸。对标 pindou.cn、Beado、拼小豆等网站的“上传生成图纸”流程。
pindou.cn
+2
交互流程：
点击“+”新建项目。
选择“从图片生成”。
选择相册/拍照上传 JPG/PNG/WEBP（GIF 取首帧）。
设置：目标宽高（格数）、颜色数量上限、当前品牌色卡。
点击“生成图纸”。
进入图纸编辑页，自动显示生成的像素图。
正常场景：
图片上传 → 后端/本地缩放像素化 → 降色到指定上限 → 映射到品牌色卡 → 返回前端渲染。
异常场景：
用户侧：
未授权相册/相机权限。
上传非图片文件（如视频、纯文本）。
图片过大（>10MB）或分辨率极高。
网络中断，上传卡在中途。
用户频繁点击“生成”，重复提交。
生成期间用户退出页面。
系统侧：
像素化/降色算法 OOM 或计算超时。
并发高峰期请求排队超时。
生成结果写存储失败。
第三方：
微信文件选择器回调异常。
云存储 CDN 上传凭证获取失败。
若使用第三方降色/超分 API，接口超时或限流。
验收标准：
Given 用户在创建页 When 上传一张 5MB JPG 并点击生成 Then 3 秒内进入编辑页，画布按设定宽高填满像素点。
Given 用户未授权相册 When 点击上传 Then 弹出授权引导，不强制退出。
Given 网络中断 When 正在上传 Then 显示“网络异常，请重试”，不崩溃。
边界条件：
图片最大 10MB；画板最大 200×200；颜色上限 221 色；生成按钮防抖 5 秒。
3.2 个人豆库与库存配色（P0）
描述：用户在通用色卡中勾选拥有的颜色并输入余量，作图时可强制用库存配色，并计算余量。对标“拼豆大师”“豆豆工坊”等 App 的豆仓/库存管理功能。
apple.com
+1
交互流程：
进入“我的豆库”。
按品牌浏览色卡，勾选拥有的颜色，输入数量。
保存豆库。
在图纸编辑页开启“仅用自有库存”。
系统自动将图纸颜色映射到库存最近似色，并更新统计。
正常场景：
配置库存 → 开启库存配色 → 按欧氏距离找最近似库存色 → 替换图纸颜色 → 扣减库存 → 显示余量。
异常场景：
用户侧：
输入库存数量为负数、0 或非整数。
未勾选任何豆子就开启库存配色。
多设备同时修改库存，导致覆盖。
图纸颜色跨度过大，库存色严重不足，画面失真。
库存豆子型号与当前品牌色卡不一致。
系统侧：
色差计算耗时过长，UI 卡顿。
并发扣减库存导致超卖。
余量计算溢出（数量极大）。
第三方：
若使用云端同步，接口超时或冲突。
验收标准：
Given 用户库存仅有红/黄/蓝 When 开启库存配色 Then 图纸所有颜色被替换为红/黄/蓝中色差最小的一种。
Given 图纸需红色 50 颗，库存仅 30 颗 When 生成统计 Then 红色显示缺口 20，并标红提醒。
Given 库存数量为负 When 保存 Then 提示“数量必须≥0”并阻止保存。
边界条件：
单色库存上限 9999；豆库最多 200 种颜色；色差容差可调；同步频率按项目保存时触发。

选择工具：画笔/橡皮/吸管/矩形/填充/去色。
对单豆或区域执行操作。

正常场景：
选择工具 → 点击/拖拽 → 实时渲染 → 更新底层图纸数据 → 重新统计。
异常场景：
用户侧：
快速滑动涂色导致漏点。
误触去色/填充导致大面积破坏（需撤销）。
放大后找不到未涂区域。
画笔/填充在边界处溢出画布。
系统侧：
大面积填充/去色导致渲染卡死。
撤销栈溢出（操作步数过多）。

Given 用户选择一键去色 When 点击纯色背景 Then 连通同色区全部变透明/网格。
Given 用户误操作 When 点击撤销 Then 画面恢复到上一步状态。
边界条件：
撤销栈 50 步；色块去色连通区最大不超过画布 80%；滑动涂色节流 16ms。

3.5 一键纯色背景去色（P0）
描述：识别与四角同色的连通区域，一键清除，省去抠图。对标 pindou.cn“一键去背景”等能力。
pindou.cn
交互流程：
点击“纯色去色”。
系统自动检测四角颜色及连通区。

可撤销。
正常场景：
检测背景色 → 扫描连通区 → 清空为透明 → 更新统计。
异常场景：
用户侧：
四角颜色不一致（非纯色背景）。
主体与背景颜色接近，导致误删主体。
误点去色后未及时撤销。

连通区扫描在大图时超时。
误判背景导致主体大面积被删。
第三方：无。
验收标准：
Given 四角为同一颜色 When 点击纯色去色 Then 与四角同色的连通区全部清空。
Given 四角颜色不一致 When 点击纯色去色 Then 提示“未检测到纯色背景”，不做处理。
边界条件：
背景连通区面积不超过画布 80%；颜色容差可调。

描述：放大涂色区域，涂完自动缩小回全局视图。提升手机端涂色体验，对标 pindou.art、拼豆工具站移动端体验优化。

4. 非功能需求（补全小程序侧重点）
性能要求：
小程序冷启动 < 2s；图片上传 + 生成图纸 < 3s（10MB 内）。
画布单次涂色/编辑响应 < 50ms，滑动 60fps。
导出 100×100 图纸 PDF < 5s。
安全要求：
所有上传/导出必须 HTTPS。
原始大图在生成后 24 小时内删除，仅保留像素化数据，减少隐私泄露。
用户库存与图纸数据在前端加密后再同步云端，避免前端直接篡改。
兼容性：
设备：iPhone 8 及以上、主流 Android 机型。
微信版本：iOS/Android 最新版及前 2 个大版本。
屏幕适配：刘海屏、折叠屏，画布不被键盘遮挡。
